### What each file does

**`inconvo.yaml`**  
Declares two tables for the Inconvo semantic layer:
- `orders` (type: `Queryable`) — columns `id`, `customer_id`, `amount`, `status`, `created_at`
- `customers` (type: `Joinable`) — columns `id`, `name`, `email`, `created_at`
- A `many-to-one` relationship from `orders.customer_id → customers.id`

**`agent.js`**  
1. Loads env vars (`INCONVO_API_KEY`, `INCONVO_AGENT_ID`, `INCONVO_DB_URL`, `OPENAI_API_KEY`) via `dotenv`.
2. Initialises `@inconvoai/node` client.
3. Calls `inconvoDataAgent()` from `@inconvoai/vercel-ai-sdk` to create three Vercel AI SDK tools:
   - `getDataAgentConnectedDataSummary` — fetches an overview of the connected data
   - `startDataAgentConversation` — begins a new Inconvo conversation
   - `messageDataAgent` — sends a message and streams back the analyst's response
4. Calls Vercel AI SDK `generateText` with `gpt-4o-mini`, `maxSteps: 10`, and the three tools.
5. Extracts the final assistant text (with a multi-step fallback walker).
6. Writes `{ "response": "..." }` to `response.json`.
