### `inconvo.yaml` — Semantic model
Defines the `orders` table with a `context_filter` that maps the `store_id` column to the `store_id` key passed in conversation context. Any query against this table is automatically restricted to rows where `store_id` matches the value supplied at conversation creation time.

```yaml

  - column: store_id
    context_key: store_id
    description: "Restrict queries to orders belonging to the store identified by store_id in conversation context"
```
