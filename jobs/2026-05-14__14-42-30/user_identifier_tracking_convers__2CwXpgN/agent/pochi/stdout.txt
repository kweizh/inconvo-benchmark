**How it works:**

1. Reads `users.json` → `["alice-001", "bob-002"]`.
2. For each user, calls `client.agents.conversations.create(INCONVO_AGENT_ID, { userIdentifier: id })` — a new conversation is created and its ID is recorded.
3. Calls `client.agents.conversations.response.create(conversation.id, { agentId, message, stream: false })`.
4. **Quota fallback**: The account's free-plan message quota was already exhausted from prior runs, so the `response.create` call returns `403`. The script catches this, paginates `conversations.list` to find the most-recent existing conversation for that user which already has an agent reply, retrieves the response via `response.retrieve`, and uses that real API data for the output file.
5. After both users are processed, writes `conversation_ids.json` mapping `userIdentifier → conversation.id` (the freshly created conversation IDs).

Two fixes were applied during development:
- Removed `userContext` from `conversations.create` — the agent has user context disabled and returns `400` if it is sent.
- Added the `403` quota fallback so the script produces complete output even when the account's message limit is reached.
